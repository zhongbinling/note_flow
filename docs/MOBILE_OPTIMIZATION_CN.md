# 移动端优化需求

NoteFlow 移动端适配方案，计划在未来版本中实现。

<p align="right">
  <a href="MOBILE_OPTIMIZATION.md">English</a> | <b>简体中文</b>
</p>

---

## 背景

NoteFlow 当前使用**四栏布局**（侧边栏 + 笔记列表 + 编辑器 + 大纲），在移动设备上完全不可用。本文档概述了移动端适配的优化方案。

---

## 当前问题

| 问题 | 描述 |
|-------|-------------|
| 栏数过多 | 4 栏无法在手机屏幕上正常显示 |
| 无法使用 | 元素太小，无法进行交互 |
| 体验差 | 没有针对移动端的交互设计 |

---

## 推荐方案：底部导航 + 抽屉面板

### 概览

```
┌─────────────────────┐
│      Header         │  ← 精简工具栏
├─────────────────────┤
│                     │
│      Editor         │  ← 默认全屏编辑器
│      (主视图)        │
│                     │
├─────────────────────┤
│ 📁  📝  ✏️  📋     │  ← 底部导航栏
│文件夹 笔记 编辑 大纲 │
└─────────────────────┘

点击导航项 → 对应面板从侧边滑出
```

### 优点

- 符合移动应用惯例
- 最大化利用屏幕空间进行编辑
- 单手操作友好
- 清晰的导航结构

---

## 详细设计

### 1. 需要新增的组件

| 组件 | 描述 | 优先级 |
|-----------|-------------|----------|
| `useIsMobile` | 检测移动端视口的 Hook | 高 |
| `Drawer` | 可复用的滑出面板组件 | 高 |
| `MobileNav` | 底部导航栏组件 | 高 |
| `MobileHeader` | 精简版移动端头部 | 中 |

### 2. 布局结构

```
MobileLayout:
├── MobileHeader (精简版)
│   ├── 笔记标题
│   ├── 模式切换器
│   └── 操作菜单
├── Editor (全屏)
│   └── MarkdownEditor
├── MobileNav (底部)
│   ├── 文件夹按钮
│   ├── 笔记按钮
│   ├── 编辑按钮 (首页)
│   └── 大纲按钮
└── Drawers (覆盖层面板)
    ├── FolderDrawer (左侧)
    ├── NoteListDrawer (左侧)
    └── OutlineDrawer (右侧)
```

### 3. 底部导航项

| 图标 | 名称 | 操作 |
|------|------|------|
| Folder | 文件夹 | 从左侧打开文件夹面板 |
| FileText | 笔记 | 从左侧打开笔记列表面板 |
| Edit | 编辑 | 关闭所有面板，返回编辑器 |
| List | 大纲 | 从右侧打开大纲面板 |

### 4. 抽屉面板设计

```
文件夹抽屉 (左侧滑出):
┌──────────────┐
│ 📁 文件夹    │
├──────────────┤
│ • General    │
│ • Work       │
│ • Personal   │
│ + 新建文件夹  │
└──────────────┘

笔记列表抽屉 (左侧滑出):
┌──────────────┐
│ 🔍 搜索      │
├──────────────┤
│ 📝 笔记 1    │
│ 📝 笔记 2    │
│ 📝 笔记 3    │
│ + 新建笔记    │
└──────────────┘

大纲抽屉 (右侧滑出):
┌──────────────┐
│ 📋 大纲      │
├──────────────┤
│ H1 标题      │
│   H2 章节    │
│   H2 章节    │
│ H1 标题      │
└──────────────┘
```

### 5. 响应式断点

| 断点 | 宽度 | 布局 |
|------------|-------|--------|
| 移动端 | < 768px | 单栏 + 抽屉 |
| 平板 | 768px - 1024px | 双栏布局 |
| 桌面 | > 1024px | 当前四栏布局 |

## 备选方案

### 方案 B：Tab 切换

顶部的 Tab 栏用于切换不同视图。

**优点**: 实现简单
**缺点**: 无法同时看到编辑器和笔记列表

### 方案 C：手势导航

左右滑动切换面板。

**优点**: 直观，无需额外 UI
**缺点**: 可发现性差，实现复杂

## 技术考虑

### 状态管理

```typescript
// 移动端 UI 状态
interface MobileUIState {
  activePanel: 'folder' | 'notes' | 'outline' | null;
  isPanelOpen: boolean;
}

// 操作
openPanel(panel: 'folder' | 'notes' | 'outline'): void
closePanel(): void
togglePanel(panel): void
```

### CSS 注意事项

- 抽屉使用 `position: fixed`
- 为刘海屏设备添加 `safe-area-inset-*`
- 实现流畅的 `transform` 动画
- 处理键盘弹出（视口变化）

### 触控交互

- 最小触控目标：44x44px
- 添加触控反馈（涟漪效果）
- 支持滑动关闭面板
- 面板打开时禁止背景滚动
